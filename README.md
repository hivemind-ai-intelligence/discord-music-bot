[Install]
WantedBy=multi-user.target

## ✅ Bug-Free Checklist

Har command, har edge case — sab manually test kiya gaya:

- ✅ **URL Playback**: YouTube, SoundCloud, direct links — sab free, koi block nehi
- ✅ **Queue Logic**: Max 50 tracks, proper FIFO, shuffle preserves all tracks, move/remove accurate
- ✅ **Connection Handling**: Timeout recovery, region mismatch, already-playing-in-other-VC protection
- ✅ **Controller**: Real-time auto-update embed, button states sync, idle mode proper
- ✅ **Autoplay**: Queue empty → related gaana auto-fetch, no infinite loop
- ✅ **Error Recovery**: Track exception → auto-skip to next, search fail → clean error message
- ✅ **Permission System**: VC validation, same-channel enforcement, admin-only 24/7
- ✅ **Inactivity**: 2-min auto-disconnect (0 if 24/7 mode)
- ✅ **Name Styles**: Per-guild settings survive bot restarts via JSON DB
- ✅ **Rate Limiting**: Har command pe proper cooldown, Discord API abuse nehi

## 🔧 Custom Commands Add Kaise Karein

Koi bhi cog file kholo (`cogs/music.py` for example):

```python
@commands.hybrid_command(name="tera_command", description="Tera command ka description")
@commands.cooldown(rate=3, per=15, type=commands.BucketType.user)
async def tera_command(self, ctx: commands.Context):
    await ctx.reply("Hello from custom command!")
```

Bot restart karo — slash commands **auto-sync** ho jaenge. Discord API automatically `/tera_command` register karega.
